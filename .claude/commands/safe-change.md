Perform a safe change following the mandatory workflow:

Task: $ARGUMENTS

## Phase 1: EXPLORE (read-only)
1. Identify which module(s) own this functionality
2. Find 2+ existing examples of similar patterns
3. List the 5-10 most relevant files

## Phase 2: PLAN
1. Provide implementation steps
2. List files to modify
3. State test strategy

## Phase 3: IMPLEMENT
1. Make minimal changes
2. Follow existing patterns
3. Add/update tests

## Phase 4: VERIFY
1. Run `cargo test`
2. Run `cargo build`
3. Summarize changes

Do NOT start editing until Phases 1 and 2 are complete.
